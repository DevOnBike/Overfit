# Changelog

All notable changes to this project are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and the project loosely adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## Versioning policy

- **`MAJOR`** aligns with the targeted .NET runtime major (currently `10`, targeting `net10.0`). A `MAJOR` bump implies a runtime target change.
- **`MINOR`** is bumped for any breaking change to the public API surface or a significant new capability surface (new loader family, new training mode, new public client).
- **`PATCH`** is bumped for backwards-compatible fixes, performance work, internal improvements, additional model architectures within an existing loader family, and documentation.

"Public API" is the surface inside `DevOnBike.Overfit.*` reachable without `InternalsVisibleTo`. Anything accessed via `InternalsVisibleTo` (`DevOnBike.Overfit.Tests`, `Benchmarks`) is **not** part of the public contract and may change between any two patches.

Pre-release suffixes (e.g. `10.1.0-beta.1`) are used for surface changes that need real-world validation before the public release. Pre-releases are pushed to NuGet with the `-beta`, `-rc`, or `-preview` SemVer suffix.

### Added

- **Three new chat architecture families** load and generate coherently in pure C# (each validated end-to-end on a real GGUF, 0 B/token decode preserved; all new arch params default to off so existing families are bit-identical):
  - **Qwen3** (`qwen3`) — per-head QK-RMSNorm before RoPE + explicit `head_dim`. Validated on Qwen3-0.6B (~70 tok/s Q8_0).
  - **Phi-3 / Phi-3.5 / Phi-4** (`phi3`) — fused QKV (`attn_qkv`) + fused gate/up (`ffn_up` = 2×dFF) split at load, LongRoPE per-dim scaling (Phi-3.5), context capped. Validated on Phi-3.5-mini **and Phi-4 14B** (Phi-4 reuses the same `phi3` arch with no longrope + a cl100k BPE tokenizer — drops in with zero new arch code; ~3.8 tok/s Q4_K_M, 0 B/token).
  - **Gemma 2** (`gemma2`) — GeGLU FFN (gelu-tanh gate), sandwich norm (`post_attention_norm` + `post_ffw_norm`), embedding ×√d_model, dual logit soft-cap (attention `tanh(s/50)·50`, final `tanh(l/30)·30`), GQA, explicit `head_dim` 256; the `(1+w)` RMSNorm offset is already baked into the llama.cpp GGUF (loaded plain). Sliding-window deferred (context capped to 4096). Validated on Gemma-2-2B-it Q4_K_M (~18–23 tok/s, thermal-dependent).
- **Docker image for the `overfit` server.** [`Sources/Cli/Dockerfile`](Sources/Cli/Dockerfile) builds the Native-AOT `overfit serve` (OpenAI-compatible HTTP server) as a self-contained native binary on a chiselled base — a **~34 MB image, no .NET runtime, no ASP.NET**. The model is never baked in (mounted at runtime; `overfit pull` can populate a volume). [`.github/workflows/docker-publish.yml`](.github/workflows/docker-publish.yml) builds and pushes to Docker Hub on a `v*.*.*` tag or manual dispatch (`DOCKERHUB_USERNAME` / `DOCKERHUB_TOKEN` secrets); the image is tagged with the `<Version>` from `Directory.Build.props` (consistent with the NuGet/tool version) + `:latest` + `:sha-<short>`. Local helpers `Sources/Cli/docker-build.cmd` + `docker-run.cmd`; hosting/model guide in [`docs/docker.md`](docs/docker.md). Validated end-to-end (image serves `/v1/chat/completions` with a mounted GGUF).
- **`overfit` as a .NET global tool.** `Sources/Cli` now packs as a tool (`PackageId` **`DevOnBike.Overfit.Cli`**, command `overfit`): `dotnet tool install -g DevOnBike.Overfit.Cli`. This is the cross-platform-convenience channel (framework-dependent IL) alongside the Native-AOT binary + Docker image (no-runtime) — both ship from the one project (`dotnet publish -r` → native; `dotnet pack` → tool). Published by `publish-nuget.yml` next to the libraries.
- **Centralised solution versioning.** A single `<Version>` in [`Directory.Build.props`](Directory.Build.props) is now the source of truth for every package (library + `.Cli` tool) — the per-project `<Version>` overrides (which had drifted to `10.0.21`/`10.0.22`) were removed. [`bump-version.ps1`](bump-version.ps1) bumps it (`patch`/`minor`/`major`/`x.y.z`, `-DryRun`) and prints the matching `git tag` to trigger the publish workflows.
- **`Microsoft.Extensions.AI.Evaluation` runs fully locally with Overfit as the judge.** Because the Overfit M.E.AI adapter is a standard `IChatClient`, Microsoft's official LLM-as-judge evaluators (Coherence / Fluency / Groundedness / …) score responses entirely in-process — no Azure, no key, no data egress. New [`Demo/EvaluationDemo`](Demo/EvaluationDemo) (validated on a real local judge: bad answers reliably flagged 1/5 `Failed`; calibrated absolute scores want a ≥7B judge) + [`docs/meai-evaluation.md`](docs/meai-evaluation.md) with honest calibration notes. Complementary to Overfit's deterministic `RagAssert` retrieval gates.
- **MCP server — plug local Overfit AI into Claude Code / Claude Desktop / IDEs** (new package **`DevOnBike.Overfit.Mcp`** + CLI command **`overfit mcp`**). A dependency-free MCP (Model Context Protocol) stdio server — typed JSON-RPC 2.0 wire contracts (`Protocol/` DTOs) serialized via a source-generated `JsonSerializerContext` (`McpJsonContext`, the same pattern as the OpenAI server's `OpenAiJsonContext`; no SDK, no reflection, no DI) so it stays Native-AOT-clean like the rest of the serving stack — AOT-verified (zero IL/trim warnings; the ILCompiler-published native `overfit.exe` serves the full handshake). Helper scripts in `Sources/Mcp/`: `mcp-register` / `mcp-unregister` / `mcp-status` / `mcp-run` / `mcp-smoke` (.cmd). Implements `initialize` (revisions 2025-06-18 / 2025-03-26 / 2024-11-05), `ping`, `tools/list`, `tools/call`; tool-execution failures flow back as `isError` results (host model can self-correct), logs on stderr, stops on stdin EOF. Built-in zero-egress tools: **`ask`** (prompt → the loaded GGUF via stateless `Complete`), **`rag_query`** (`--rag-dir`: grounded answers with `[n]` citations over local `.txt`/`.md`, chunked + embedded with the chat model's OWN multilingual embeddings — no second model), **`transcribe`** (`--whisper-model`, lazy-loaded: WAV/MP3 → text, pure-C# Whisper). Register: `claude mcp add overfit -- overfit mcp <model.gguf>`. Protocol layer covered by 12 fast model-free tests; validated end-to-end against a real `overfit.exe` over stdio (Qwen3-0.6B + whisper-tiny: handshake, "Paris", cited RAG answer, verbatim JFK transcript) **and inside real Claude Code** (`claude mcp add` → `✔ Connected` → a live session drove all three tools, including a Polish `rag_query` over a Polish document — the chat-model embeddings are multilingual in practice). Docs: [`docs/mcp.md`](docs/mcp.md).
- **`min_p` sampling on `/v1/chat/completions`** (llama.cpp-server extension, supported by most local-AI clients): keep tokens with probability ≥ `min_p` × P(top) — quality-safer than `top_p` on small models. Maps to the existing `SamplingOptions.WithMinP`; takes precedence over `top_p` when both are sent.
- `OverfitClient.LoadGguf(path)` — turnkey one-line facade for in-process GGUF chat (Qwen / Llama / Mistral / Mixtral / Qwen-MoE / GPT-2 family). Auto-detects chat template, tokenizer, and stop sequences.
- `OverfitClient.LoadGguf` now loads a **bare `.gguf` with no sibling tokenizer files** — it falls back to the tokenizer embedded in the file's `tokenizer.ggml.*` metadata via the new `GgufEmbeddedTokenizer` (an `ITokenizer` adapter over `GgufTokenizer`). Sibling `tokenizer.json` / `vocab.json`+`merges.txt` are still preferred when present (unchanged). Validated by loading the Polish **Bielik-4.5B-v3.0-Instruct** Q8_0 GGUF (arch `llama`, SPM tokenizer, ChatML template — all auto-detected) and generating coherent Polish, 0 B/token.
- `OverfitClient.LoadPretrained(modelDir)` — same turnkey facade for an unpacked HuggingFace directory (`model.safetensors` [+ shards] + `config.json` + tokenizer), via `SafetensorsLlamaLoader` — no GGUF conversion step. Detects the chat template from `tokenizer_config.json`. Both factories take an optional `SamplingOptions sampling` (default greedy). The Local Agent demo now accepts a `*.gguf` file **or** a safetensors directory for `ModelPath` / `OVERFIT_MODEL_DIR`.
- `JsonGrammarConstraint(tokenizer, requireObject: true)` — optionally force the JSON root to be an **object** (until the opening `{`, only whitespace or a `{`-leading token is allowed). Prevents a model from satisfying "valid JSON" with a bare string/number/array — e.g. a quoted string that merely *contains* JSON, which some models (observed on Bielik) emit for JSON-mode. Default stays "any well-formed JSON value". The Local Agent demo's `/chat/json` and `/decision/refund` use it.
- `ChatSession.Complete(...)` / `OverfitClient.Complete(...)` — stateless one-shot generation that renders only `[system turns] + this user turn`, generates, and records nothing in the conversation history. For task-style calls (tool routing, JSON mode, RAG answers) this avoids both inheriting prior turns and accumulating across calls, so each call prefills only its own minimal prompt. `Send(...)` remains the multi-turn conversational path. In the Local Agent demo, `/agent`, `/chat/json` and `/rag/query` now use it — on Qwen2.5-3B an interleaved (no-reset) run dropped `/agent` 18.8 s → 5.6 s and `/chat/json` 18.9 s → 2.5 s (they had been re-prefilling the whole accumulated conversation, including the ~418-token RAG context, every call).
- Per-tool **argument schema enforcement** in `ToolCallConstraint`: a `ToolDefinition` may now declare ordered `ToolParameter`s (name + `ToolParameterKind` String/Number/Integer/Boolean). When present, constrained decoding forces the `arguments` object to carry exactly those keys, in order, each with a value of the declared type — the model cannot invent a key name, omit a required argument, or emit the wrong value type. Tools with no declared parameters keep the previous "any well-formed JSON" behaviour (backward compatible).
- BERT-family sentence embeddings with bit-parity vs HuggingFace / PyTorch: `SentenceEmbedder.ForMiniLm`, `ForBgeEnV15`, `ForE5` (validated cosine ≈ 1.0 on real weights).
- ReAct agentic loop driver (`ReActAgent`) over `ChatSession` + `ToolCallConstraint`, with auto-registered `finish` tool.
- Sliding-window eviction in `ChatSession` for long-running conversations (`new ChatSession(slidingWindow: true)`).
- Mixture-of-Experts inference: Qwen1.5-MoE and Mixtral-8x7B (Q4_K_M and Q8_0), validated coherent on real models.
- `DataParallelTrainer` — N replicas + gradient averaging, model-agnostic.
- Gradient checkpointing for training memory (24× reduction on 12-layer GPT-1).
- CRNN / CTC training stack: `Crnn` IModule facade, `CtcLoss` (NLL + finite-difference verified), `CtcDecoder` (greedy), `TransposeLastTwo` graph op.
- Convolution training: padding / stride / bias with correct backward (previously VALID / no-bias only); finite-difference verified.

- ONNX import: linear topology (`OnnxImporter.Load`) and DAG with skip connections (`OnnxGraphImporter.Load`). External `.data` sidecar files resolved automatically. Hand-rolled protobuf parser (no `Google.Protobuf` dependency).

### Performance

- **CPU decode throughput sprint (Bielik-4.5B Q4_K_M): 12.55 → ~17 tok/s (+36%), bit-identical, full suite green.** A stack of independently-measured, bit-identical levers, each A/B-validated best-of-N on a quiet box: (1) **decode worker-cap** — `OverfitParallel.DecodeMaxWorkers` (default `min(WorkerCount, 10)`, env `OVERFIT_DECODE_WORKERS`) caps the FFN matmul dispatch, since the small per-token matmuls were dispatch-overhead-bound, not bandwidth-bound, and 32 workers oversubscribed one matrix (+13%); (2) **fused SwiGLU gate+up** — `Q4KDotKernel.ProjectGateUpParallel` quantizes `hidden` to Q8_K once and runs both projections in one dispatch (+6%); (3) **decode spin-pool** — `OVERFIT_DECODE_POOL` routes decode dispatches to a cap-sized spinning worker pool (no per-op semaphore park/wake), like llama.cpp `--poll` (+6%; since made default-ON, and the busy-spin replaced with spin-then-park — see Fixed); (4) **SIMD activation-quantize** — `QuantizeActivationQ8KAvx2` (llama.cpp's is scalar — a net edge) (+4.6%); (5) **SIMD RoPE** — `RopeKernel.ApplySplitHalfAvx2` / `ApplyAdjacentAvx2` (−15% attention); (6) **SIMD cached-attention** — `CachedAttentionKernel.ComputeSingleHead` weighted-V sum (bit-identical) + reassociated score-dot (+12% long-context). Same-file gap to llama.cpp/LLamaSharp narrowed from ~1.33× to **~1.13×**. **Honest standing: still ~1.13× behind on raw decode (NOT parity); single-stream CPU decode is DRAM-bandwidth-bound** — the residual gap is memory-access efficiency, not ALU, so closing it further needs structural (not kernel-ALU) work. Overfit keeps its −36 % working-set + ~1 B/token edge throughout.
- **`DecodeProfiler`** — public, off-by-default, zero-overhead-when-off per-component decode profiler (Attention / FFN / LM-head / Sampler / whole-token). AOT-safe (no LINQ/reflection). Drove the sprint: confirmed FFN ≈ 78 % of decode, killed measured-worthless optimizations (streaming-attention, argmax-fusion, sampler-opt).
- **Repacked 8×8 K-quant GEMV kernels (AVX2, opt-in `OVERFIT_REPACK_GEMV=1`).** `Q4KGemvKernel` / `Q4KRepack` — faithful port of llama.cpp `ggml_gemv_q4_K_8x8_q8_K` (`block_q4_Kx8`, 8 weight rows interleaved → 8 outputs in SIMD lanes, no per-row horizontal reduction); validated `maxRelDiff ≈ 1.4e-6` vs the 1-row kernel; wired into the decode FFN gate/up projections (+6.8 %). `Q6KGemvKernel` / `Q6KRepack` — a **novel** AVX2 Q6_K 8×8 GEMV (llama.cpp ships no x86 Q6_K 8×8, only ARM NEON + a scalar generic), built on `maddubs` with the `−32` bias folded via the `m32s` trick; validated ~1.7× the 1-row kernel single-thread and bit-close to a scalar oracle across sizes/seeds. Library assets — the Q6_K kernel is intentionally **not** wired into decode (the down-projection is DRAM-bandwidth-bound, so it measured no end-to-end gain and the repacked copy costs RAM).
- **MNIST / CNN training: full 60k × 5-epoch train ~6.1 s → ~2.1 s (2.7×) at identical final loss.** Three measured levers: (1) the existing `DataParallelTrainer` applied ×8 with the **linear LR rule** (lr×R beat √R on final loss) + batch 128/replica; (2) **AVX2 MaxPool2D pool=2 kernel** — the scalar branchy horizontal-max/argmax tail replaced with shuffle/permute deinterleave + GE-mask blends + exact INT32 index math, bit-identical **values and argmax indices** (indices drive the backward scatter), op −26% (gated by fast `MaxPoolPool2Avx2ParityTests`); (3) suppress-guard fixes (see Fixed). Full per-op audit — including measured dead-ends (int8 training forward 0.61×, VNNI ≈0, ReLU-bwd Vector512 ≈0) and why 16 physical cores are already saturated — in [`docs/mnist-cnn-training-audit.md`](docs/mnist-cnn-training-audit.md).
- **Q8 KV-cache attention kernel (Phase 1).** `CachedAttentionKernel.ComputeSingleHeadQ8` + `Q8KvQuant` — attention over per-position symmetric-int8 K/V (one F32 scale per cached vector, query stays F32), the foundation for a ~4×-smaller KV cache (longer context in the same RAM). Validated cosine ≈ 0.99999 vs the F32 attend across seq 8–1500 / headDim 64–128. Library asset; the int8 cache storage + write path is a follow-on.

### Fixed

- **Decode spin-pool burned 100% CPU at idle** (field report: a serving container on a 16-core laptop pegged all cores while waiting for requests — the default-ON pool's workers pure-hot-spun between tokens). The pool is now **spin-then-park**: workers hot-spin ~1–2 ms past the last dispatch (so a token in flight never parks — decode throughput unchanged, 0 B/token preserved), then park on a semaphore; the wake protocol is missed-wake-free (worker registers before its final generation re-check, dispatcher bumps the generation before reading the parked count). Idle process: ~10 effective cores → **~0**. The `OVERFIT_DECODE_POOL=0` workaround is no longer needed (it costs ~28% decode on small models). Gated by `DecodePoolIdleBurnTests` (`[LongFact]`: < 1 effective core during a 3 s post-decode idle window).
- **Five training graph-ops ignored `SuppressParallelismOnCurrentThread`** (SDPA backward, BatchNorm2D backward ×2, LSTM forward + backward used raw `Parallel.For`), so inside `DataParallelTrainer` replicas they oversubscribed the box (R replicas × N threads). All five now dispatch via `OverfitParallel.For`, which inlines under suppression — the audit measured suppression as load-bearing (disabling it: 1.8× slower).
- **`BertEncoder` arena `OutOfMemoryException` at full sequence length.** The arena was sized from an activation-data estimate, but the encoder's `ComputationGraph` holds the whole forward tape live (every layer's activations + a per-node gradient buffer) until `Reset()` — so the real peak at `T = MaxSequenceLength` (256) is ~2.5× the data estimate. The previous +25% margin was only ever enough because short sentences (and the MiniLM/BGE/E5 parity checks) never reached full length; a full-length 256-token passage — exposed by Polish RAG chunks, which sub-word heavily — threw `NativeBuffer exhausted`. Arena now sized at 2.5× the estimate. Guarded by `MiniLmFullLengthTests` (`[LongFact]`).
- **Native-AOT guard (CI) failed to publish.** Two real issues plus a tooling mismatch: (1) `BytePairEncoder.ParseVocabJson` used reflection-based `JsonSerializer.Deserialize<Dictionary<string,int>>` — `RequiresUnreferencedCode` + `RequiresDynamicCode` (IL2026 / IL3050), which genuinely breaks under Native AOT; rewritten to a reflection-free `JsonDocument` parse. (2) `ArrayPoolEventSource` triggered IL2026 on `EventSource.WriteEventCore`; suppressed with justification (the calls pass only primitive `EventData*` payloads, so they are trim/AOT-safe). Also enabled the nullable **annotation** context on `Main` (`<Nullable>annotations</Nullable>`) so its pervasive `?` reference annotations stop emitting CS8632 (which the guard's `TreatWarningsAsErrors` turned into errors), removed an unreachable-code path (CS0162) in `FastTensor`, and scoped the CI guard so it promotes only trim/AOT (IL) warnings to errors — the `IDisposableAnalyzers` (IDISP*) and XML-doc (CS15xx) warning families, which are intentional and unrelated to AOT, no longer fail the job.
- **Constrained decoding (tool calls / JSON mode) ran to `MaxNewTokens` instead of stopping when the structure completed.** Once a `ToolCallConstraint` / `JsonGrammarConstraint` closed its root value, the only still-unmasked tokens were trailing whitespace, and greedy kept sampling them up to `MaxNewTokens` — wasting ~`MaxNewTokens` decode steps per call and appending a whitespace tail to the output. `ChatSession.Generate` now breaks as soon as `constraint.IsComplete` is true. Measured on Qwen2.5-3B: a guaranteed-JSON reply went 16.8 s → 2.4 s, the three demo agent/JSON calls went from 768 generated tokens to 74, and the whitespace tail is gone.
- **Tool-call `arguments` keys were unconstrained, so a model could pick the right tool but emit wrong/missing argument names.** Observed live: the model routed to `create_ticket` correctly but produced `{"email", "priority", "description"}` instead of the handler's `{"customerEmail", "subject", "priority"}`, so dispatch failed. Now that a tool can declare a parameter schema (see Added), the demo's tools do, and the constraint forces the exact keys/order/types — the same request now yields a valid, dispatchable call. Guarded by fast model-free `ToolCallConstraintTests` (exact keys, wrong-key rejection, wrong-type rejection, order enforcement, missing-key rejection).

- **Qwen2/Qwen2-MoE GGUF: wrong RoPE rotation convention corrupted multi-token attention.** The GGUF loader applied **adjacent-pair** RoPE `(x[2i], x[2i+1])` to Qwen2 Q/K weights, but llama.cpp stores them in the HF/NEOX **split-half** layout `(x[i], x[i+d/2])` (unlike Llama/Mistral, which llama.cpp permutes to adjacent at conversion). The mismatch is invisible at position 0 (identity rotation) but corrupts every later position — attention collapsed onto the current token, so any real chat with a system message (≥ a handful of tokens) produced degenerate, space-less output (`"France'scapitalistouredisParis."`) while short prompts looked fine. The HF-safetensors loader already permuted Q/K for adjacent rope (validated), so only the GGUF path was affected, and it was masked because the integration tests used a 3-token prompt + a `[LongFact]` parity check. Added `GPT1Config.RopeSplitHalf` (set for `qwen2`/`qwen2moe` GGUF), threaded through `RopeTable` → `RopeKernel`. Verified against HF/transformers loading the identical GGUF: Overfit now matches (`"The capital of France is Paris."`), and the full in-process RAG + tool-calling + JSON agent stack works correctly on Qwen2.5-3B. Guards (all fast / CI-runnable): `RopeConventionTests` — model-free kernel checks (relative-position invariance, adjacent≠split-half, position-0 identity) plus tiny (~40 KB) `tiny-qwen2.gguf` / `tiny-llama.gguf` fixtures (`Tests/test_fixtures/gguf/`, regenerate via `Scripts/make_tiny_gguf_fixtures.py`) that load through the real loader and assert the arch→`RopeSplitHalf` wiring; plus `BatchedPrefillParityTests.Engine_GeneratesCoherentText_ForLongSystemPrompt` (`[LongFact]`, real-model integration guard).
- **`BertEncoder` arena under-sizing → `OutOfMemoryException` on longer inputs.** The autograd arena was sized as `NumLayers · T · dFF · k`, which ignored the attention-scores term (`heads · T²`) that dominates at the sequence-length limit — so embedding a paragraph-length text (near the 256-token default) threw `NativeBuffer exhausted`, violating the encoder's contract that inputs up to `MaxSequenceLength` are valid. Re-derived the arena from an explicit per-layer tape estimate (`2·heads·T² + 2·T·dFF + 16·T·d`, +25% headroom). Surfaced while wiring in-process RAG over real documents.
- **Native AOT guard now actually verifies Native AOT.** Previously the `aot-guard` CI job ran `dotnet publish` on `Sources/Main` (a class library), which does not invoke ILCompiler — only an executable can be AOT-compiled. The check was symbolic and would not catch trim/AOT-incompatible code added to the library. Replaced with a thin `Sources/AotSmokeTest` console exe that references `DevOnBike.Overfit`; the CI job now publishes the smoketest under `-p:PublishAot=true -p:TreatWarningsAsErrors=true` on Ubuntu (with `clang` + `zlib1g-dev` installed) and executes the produced native binary as a runtime smoke check.
- `devskim` and `checkmarx-one` GitHub Actions workflows now target the `main` branch — previously they targeted the non-existent `master`, so scans never triggered automatically.
- GPT-2 loader peak-RAM regression: lazy chunk streaming reduces peak load RAM from ~2× steady-state to ~1× steady-state.
- RoPE row-permutation bug in `SafetensorsLlamaLoader` (HF rotate-half vs GGUF adjacent-pair on q/k) — validated coherent on real Qwen2.5-0.5B.
- Mixture-of-Experts `norm_topk_prob` handling is now architecture-aware (`qwen2moe` = raw softmax, Mixtral = renormalised) — verified coherent on full 24-layer Q8_0 run.

## [10.0.21] - 2026-06-05

### Added

- **`overfit` global CLI** — a single self-contained Native-AOT binary (`pull` / `list` / `chat` / `serve`). `pull` resolves a model alias, a HuggingFace `owner/repo`, **or a direct `https` URL**; downloads stream to a local store (`~/.overfit/models`) with **SHA-256 verification** (HuggingFace LFS `oid`, or a sibling `{url}.sha256`) and **HTTP-Range resume** of interrupted transfers. Set `HF_ENDPOINT` to route through a HuggingFace mirror (e.g. `https://hf-mirror.com`) when `huggingface.co` is blocked.
- **`DevOnBike.Overfit.Server`** (new NuGet package) — a dependency-free, OpenAI-compatible HTTP server (`OverfitOpenAiServer`) built on `System.Net.HttpListener` + `System.Text.Json` source-gen, no ASP.NET Core, Native-AOT friendly. Exposes `/v1/chat/completions` (streaming SSE + non-streaming, with `response_format` JSON / JSON-Schema constrained output), `/v1/models` and `/health`. Backs `overfit serve`; the ASP.NET demo shares the same DTOs + request→runtime mapping so the wire contract can't drift.
- **`DevOnBike.Overfit.Exceptions`** — a domain-exception hierarchy: `OverfitException` (base) with `OverfitFormatException` (malformed/inconsistent model or data file) and `OverfitRuntimeException` (invalid state / unsupported operation). Catch `OverfitException` to handle any Overfit-domain failure distinctly from system and programming errors.

### Changed

- Domain throw sites across the runtime moved onto the `OverfitException` hierarchy (`InvalidDataException` → `OverfitFormatException`; `InvalidOperationException` / `NotSupportedException` → `OverfitRuntimeException`). The `ArgumentException` family and `ObjectDisposedException` are intentionally unchanged (they signal programmer/argument errors, not domain failures).
- `DevOnBike.Overfit` no longer emits a legacy `*.symbols.nupkg` — debug symbols are embedded in the assembly (`DebugType=embedded`).
- **Source Link** (GitHub) enabled for all published packages, so consumers can step into the exact source from the embedded PDBs.

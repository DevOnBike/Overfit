# Overfit ONNX Import — Work-in-Progress Pack

This zip contains the in-progress ONNX import feature for [DevOnBike/Overfit](https://github.com/DevOnBike/Overfit).

## What's inside

```
ONNX_IMPLEMENTATION_PLAN.md         ← Read this first
Sources/Main/Onnx/                  ← Core ONNX parsing & import
├── OnnxImporter.cs                 ← public entry point: OnnxImporter.Load(path)
├── OnnxModel.cs                    ← domain types (OnnxModel, OnnxNode, OnnxTensor, ...)
├── OnnxProtoParser.cs              ← protobuf → OnnxModel
├── Protobuf/
│   └── ProtoReader.cs              ← minimal protobuf wire format reader
└── Operators/
    └── _TODO.cs                    ← put operator handlers here

Sources/Main/DeepLearning/
├── MaxPool2DLayer.cs               ← new IModule for MaxPool (was function in TensorMath)
├── GlobalAveragePool2DLayer.cs     ← new IModule for GAP
└── FlattenLayer.cs                 ← new IModule for [batch, ...] → [batch, prod(...)]

test_fixtures/
└── generate_fixture.py             ← Python script to generate test ONNX + reference data
```

## How to integrate into Overfit repo

```bash
# From the Overfit repo root:
unzip overfit-onnx-pack.zip -d /tmp/onnx-pack
cp -r /tmp/onnx-pack/Sources/* Sources/
mkdir -p Tests/test_fixtures
cp /tmp/onnx-pack/test_fixtures/generate_fixture.py Tests/test_fixtures/

# Generate test data once (requires PyTorch):
cd Tests/test_fixtures
python generate_fixture.py
cd ../..

# Build to confirm core compiles (operator handlers TODO):
dotnet build Sources/Main/Main.csproj
```

The core (parser, model types, importer skeleton, layer wrappers) builds and is ready to use.
The `Operators/_TODO.cs` is just a placeholder — replace with actual operator handlers per the plan.

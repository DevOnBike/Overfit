N:System.Linq; LINQ is banned for performance reasons. Use for/foreach loops and Span<T>.
N:System.Reflection; Reflection is slow and frequently breaks Native AOT. Use delegates, static typing, or Source Generators.
T:System.Linq.Expressions.Expression; Expression trees use Reflection.Emit, which throws under Native AOT (no JIT). Use delegates and 'switch'.
T:System.Activator; Activator.CreateInstance is slow and requires type metadata under AOT. Use explicit 'new' or factories (Factory Pattern).
M:System.Array.Copy(System.Array,System.Array,System.Int32); Use Span<T>.CopyTo — it is as fast or faster (benchmarked) and has memmove semantics for overlapping copies.
M:System.Array.Copy(System.Array,System.Array,System.Int64); Use Span<T>.CopyTo — it is as fast or faster (benchmarked) and has memmove semantics for overlapping copies.
M:System.Array.Copy(System.Array,System.Int32,System.Array,System.Int32,System.Int32); Use Span<T>.CopyTo (source.AsSpan(srcIdx,len).CopyTo(dst.AsSpan(dstIdx,len))) — as fast or faster, memmove-safe.
M:System.Array.Copy(System.Array,System.Int64,System.Array,System.Int64,System.Int64); Use Span<T>.CopyTo (source.AsSpan(srcIdx,len).CopyTo(dst.AsSpan(dstIdx,len))) — as fast or faster, memmove-safe.
P:System.Buffers.ArrayPool`1.Shared; Use PooledBuffer<T> (Tensors/PooledBuffer.cs) — the project's single audit point over ArrayPool<T>.Shared. Two modes: (1) scoped `using var buf = new PooledBuffer<T>(n, clearMemory: false);` for ref-struct lifetimes (including inside Parallel.For lambdas); (2) class-lifetime `PooledBuffer<T>.RentArray(n)` / `PooledBuffer<T>.ReturnArray(arr)` for cases where the using scope doesn't fit (FastTensor, TensorStorage, loaders). 2026-05-29 PoolComparisonBenchmark: .Shared is 3× faster single-thread AND ~3000× faster than ArrayPool.Create-based pools under multi-thread contention (TLS per-CPU caches vs lock-per-bucket). Wrapper overhead is ~0%.
